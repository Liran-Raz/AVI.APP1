# AVI.APP — מדריך בנייה והרצה של אפליקציית המובייל

מדריך צעד-אחר-צעד להפיכת הקוד לאפליקציה מותקנת על iPhone/Android ולהגשה לחנויות.
כתוב עבור מי שאינו מפתח — כל פקודה מוסברת. הפרויקט הנייטיבי נמצא בתוך `web/` (Capacitor 8).

> ‏DEV-025 · ענף `feat/dev-025-m1-web`. המצב: כל קוד M1 מוכן ונבדק; נותרו הגדרה, בנייה ובדיקת-מכשיר.

---

## 0. הרעיון הגדול — למה זה חוסך לך המון

האפליקציה במצב **Hosted**: ה-WebView טוען את האתר החי `https://www.aviapp1.com`, לא עותק ארוז.
המשמעות המעשית:

- **כל שינוי בקוד-הווב** (מסכים, לוגיקה, תיקונים) → פשוט מתפרסם ל-Vercel כרגיל, והאפליקציה
  **מתעדכנת מיידית** אצל כל המשתמשים. **בלי בנייה מחדש ובלי עדכון-חנות.** זה היתרון הגדול.
- **בנייה מחדש של האפליקציה + עדכון-חנות** נדרשים **רק** כששינית משהו **נייטיבי**: אייקונים,
  splash, פלאגינים, הגדרות, או ה-deep-links.

## 2. הגדרת Supabase (2 דקות — חובה להתחברות Google)

בלי זה, התחברות Google בנייטיב תיכשל (מייל/סיסמה יעבוד בכל מקרה).

1. היכנס ל-Supabase Dashboard → הפרויקט → **Authentication** → **URL Configuration**.
2. תחת **Redirect URLs**, לחץ **Add URL** והוסף בדיוק:
   ```
   com.aviapp1.app://auth/callback
   ```
3. **Save**. (אל תמחק את כתובת ה-`https://www.aviapp1.com/...` הקיימת — שתיהן נחוצות.)

## 3. הכנת הפרויקט (לפני כל בנייה)

בטרמינל, מתיקיית הפרויקט:

```bash
cd web
npm install        # מתקין את התלויות (כולל Capacitor)
npx cap sync       # מסנכרן קונפיג + פלאגינים אל הפרויקטים הנייטיביים
```

> `cap sync` לא בונה את הווב (מצב hosted) — הוא רק מעדכן את `ios/` ו-`android/`.
> הרץ אותו אחרי כל `git pull` או שינוי נייטיבי.

## 4. בניית Android (`.aab` להגשה ל-Google Play)

**דרך מומלצת — דרך Android Studio (גם יוצרת את מפתח-החתימה):**

```bash
cd web
npx cap open android      # פותח את הפרויקט ב-Android Studio
```

ב-Android Studio:
1. תן לו לסיים "Gradle sync" (הורדה ראשונה איטית — צריך אינטרנט).
2. תפריט **Build → Generate Signed Bundle / APK → Android App Bundle**.
3. **Create new…** keystore (בפעם הראשונה) — בחר סיסמה ושמור את הקובץ.
   ⚠️ **גבֵּה את קובץ ה-keystore ואת הסיסמאות במקום בטוח.** הוא **קבוע** — אובדנו =
   לא ניתן יותר לעדכן את האפליקציה ב-Play.
4. בחר **release** → **Finish**. הפלט: `app-release.aab` (זה מה שמעלים ל-Google Play).

**להתקנה מהירה על מכשיר-בדיקה** (בלי חנות): חבר טלפון אנדרואיד ב-USB (Developer Mode פעיל),
ולחץ ▶ Run ב-Android Studio.

---

## 5. בניית iOS (חובה Mac + Xcode)

```bash
cd web
npx cap open ios          # פותח את App.xcodeproj ב-Xcode
```

ב-Xcode:
1. תן ל-Xcode לפתור את ה-Swift Packages (אוטומטי, פעם ראשונה איטית).
2. בחר את ה-target **App** → לשונית **Signing & Capabilities** → סמן **Automatically manage
   signing** ובחר את ה-**Team** שלך (חשבון ה-Apple Developer).
3. **להתקנה על iPhone לבדיקה:** חבר את המכשיר, בחר אותו למעלה, ולחץ ▶ Run.
4. **להגשה לחנות:** תפריט **Product → Archive** → **Distribute App → App Store Connect**.

## 7. מה אחרי הבדיקה (הגשה לחנויות — M1-D)

1. **קבצי deep-link (`.well-known`)** — לשדרוג ה-OAuth ל-Universal/App Links (מאובטח יותר
   מ-scheme). דורש את ה-**Apple Team ID** ואת **טביעת-האצבע SHA256** של מפתח-החתימה ב-Android —
   שניהם נוצרים רק אחרי שיש לך חשבונות + keystore. תגיד לי כשיש, ואשלים אותם.
2. **נכסי-חנות:** צילומי-מסך, תיאור, מדיניות-פרטיות (הקישור: `https://www.aviapp1.com/privacy`).
3. **שאלון-פרטיות** בכל חנות (איזה מידע נאסף) — עמודי הפרטיות/תנאים כבר מכסים את התוכן.
4. **הגשה + ביקורת.** אפל עשויה לשאול על "minimum functionality" (סעיף 4.2) — ממותן ע"י
   היות זה כלי-עסקי מלא + (ב-M2) התראות-push נייטיביות. לתקצב סבב-ביקורת אחד.
